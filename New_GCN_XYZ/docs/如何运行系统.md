# 如何运行AGV调度系统

## 系统概述

这是一个完整的GCN引导的在线AGV调度系统，实现了你的毕业设计思路：
- ✅ GCN热力图构造（无监督学习）
- ✅ 充电约束管理
- ✅ 时间窗约束
- ✅ 在线动态调度
- ✅ 使用robot_data真实数据
- ✅ 与AnyLogic文件交换接口

## 快速启动

### 方法1：使用启动脚本（推荐）

1. 双击运行 `RUN_SYSTEM.bat`
2. 脚本会自动：
   - 检查并安装所需的Python包（numpy, openpyxl）
   - 启动AGV调度系统
   - 等待AnyLogic发送订单

### 方法2：手动运行

1. 打开命令提示符（CMD）或PowerShell
2. 安装依赖包（如果还没安装）：
   ```bash
   pip install numpy openpyxl
   ```

3. 运行系统：
   ```bash
   python "d:\1nottingham\Year4a\FYP\hospital_main_copy\-Heatmap-VRPTW\New_GCN_XYZ\anylogic_file_bridge.py"
   ```

## 系统启动流程

系统启动时会执行以下步骤：

```
[1/4] 加载robot_data数据
  - 从robot_order.xlsx加载订单
  - 从robot_node.xlsx加载节点
  - 从robot_edge.xlsx加载边
  - 提取充电站位置

[2/4] 构建图并生成GCN热力图
  - 构建节点映射
  - 训练GCN模型（500轮无监督学习）
  - 生成热力图矩阵

[3/4] 初始化在线调度器
  - 创建9个AGV
  - 设置充电站
  - 启用充电和时间窗约束

[4/4] 系统就绪
  - 等待AnyLogic发送订单
  - 监控输入目录
```

## 常见问题

### 问题1：ModuleNotFoundError: No module named 'numpy'

**解决方案**：
```bash
pip install numpy openpyxl
```

### 问题2：PyTorch import error

**解决方案**：系统会自动使用numpy fallback模式，不需要PyTorch也能运行。

### 问题3：openpyxl not found

**解决方案**：
```bash
pip install openpyxl
```
或者系统会使用fallback数据（测试数据）。

### 问题4：系统启动后没有反应

**原因**：系统正在等待AnyLogic发送订单。

**解决方案**：
1. 检查 `anylogic_bridge/input/` 目录是否有订单文件
2. 或者手动创建测试订单文件

## 测试系统

### 创建测试订单

在 `anylogic_bridge/input/` 目录下创建 `test_order.json`：

```json
{
  "id": "TEST_001",
  "start_x": 0.0,
  "start_y": 0.0,
  "start_z": 0,
  "end_x": 100.0,
  "end_y": 100.0,
  "end_z": 1
}
```

系统会自动：
1. 检测到订单文件
2. 加载订单
3. 执行调度算法
4. 输出路由到 `anylogic_bridge/output/routes.json`
5. 删除输入文件

## 系统配置

配置文件位于 `core/config.py`，可以修改：

- `AGV_NUM = 9` - AGV数量
- `BATTERY_CAPACITY = 10000.0` - 电池容量
- `ENABLE_CHARGING = True` - 启用充电约束
- `ENABLE_TIME_WINDOWS = True` - 启用时间窗约束
- `LAMBDA_DIST = 0.5` - 距离权重
- `GAMMA_HEAT = 200.0` - 热力图权重

## 停止系统

按 `Ctrl+C` 停止系统。系统会显示最终统计信息。
