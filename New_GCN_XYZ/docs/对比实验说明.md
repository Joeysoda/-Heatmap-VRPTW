# 算法对比实验说明 / Algorithm Comparison Guide

## 📋 概述 / Overview

本实验框架用于比较GCN引导的启发式算法与三个基线算法的性能，基于中期报告Section 3.3的规范实现。

This framework compares the GCN-Guided Heuristic with three baseline algorithms, implemented according to Section 3.3 of the interim report.

## 🎯 对比算法 / Algorithms Compared

### 1. GCN-Guided Heuristic（我们的方法）
- **策略**: 使用GCN生成的热力图引导两阶段插入启发式
- **特点**: 
  - Stage 1: 优先分配空闲AGV（负载均衡）
  - Stage 2: 优化插入位置（考虑距离、负载和热力图）
- **评分函数**: `Score = λ_dist·Δdist + λ_load·Load_new - γ·Δheat`

### 2. Best Fit（基线1）
- **策略**: 贪心算法，选择导致全局成本增加最小的AGV和位置
- **特点**: 关注局部效率，每次选择最优插入
- **评分标准**: 纯距离增量

### 3. First Fit（基线2）
- **策略**: 轮询分配，按顺序分配订单到AGV（1, 2, ..., N, 1...）
- **特点**: 确保任务数量均衡，但忽略空间效率
- **评分标准**: 在选定AGV内找最优位置

### 4. Nearest Neighbor（基线3）
- **策略**: 空间贪心，选择当前最后位置最接近订单起点的AGV
- **特点**: 最小化即时设置距离，可能导致负载不平衡
- **评分标准**: 距离最近

## 📊 评估指标 / Evaluation Metrics

1. **Makespan**: 所有AGV中的最大完成时间
2. **Total Travel Time**: 所有AGV的总行驶时间
3. **Total Cost**: 全局目标函数值（结合Makespan和局部成本）
4. **Improvement %**: GCN相对于最佳基线的改进百分比

## 🔧 自定义实验 / Customization

### 修改测试规模
编辑 `run_comparison.py` 中的 `test_scales` 列表：
```python
test_scales = [50, 100, 200, 500, 1000, len(all_orders)]
```

### 修改算法参数
编辑 `core/config.py` 中的参数：
```python
LAMBDA_DIST = 1.0    # 距离权重
LAMBDA_LOAD = 0.5    # 负载权重
GAMMA_HEAT = 2.0     # 热力图权重
```

## 📖 结果解读 / Interpreting Results

### 改进百分比计算
```
Improvement % = ((Best_Baseline_Cost - GCN_Cost) / Best_Baseline_Cost) × 100
```

- **正值**: GCN优于基线（越大越好）
- **负值**: GCN劣于基线
- **0**: 性能相当

### 预期结果
根据中期报告的初步结果：
- 小规模实例（20-73订单）: 改进27%-37%
- 中大规模实例（100+订单）: 改进10%-36%
- 平均改进: 约25%

## ⚠️ 注意事项 / Notes

1. **运行时间**: 完整实验（6个规模）可能需要10-30分钟
2. **内存需求**: 大规模实验（2328订单）需要足够内存
3. **GCN训练**: 每次运行都会重新训练GCN（500 epochs）
4. **图表生成**: 需要matplotlib库，如果没有安装会跳过图表生成

## 🐛 故障排除 / Troubleshooting

### 问题1: matplotlib不可用


### 问题2: 内存不足
**解决**: 减少测试规模，只测试较小的订单数量

### 问题3: Python路径错误
**解决**: 修改批处理文件中的Python路径
```batch
D:\Python311\python.exe -> 你的Python路径
```
